# Локальный STT-сервис (faster-whisper)

Самодостаточный сервис для распознавания русской речи
**на вашем сервере, без внешних API**.

Сервис можно использовать в двух режимах:
- как отдельный тестовый контур (`docker-compose.stt.yml`);
- как часть основного деплоя через `docker-compose.yml` (автоподъём вместе с приложением).

- Аудио обрабатывается в памяти, на диск не пишется (кроме кэша модели).
- Batch-режим: загрузили запись → получили текст.
- Не логирует содержимое распознавания (только тех. метрики).

## Запуск отдельно (тестовый контур)

```bash
# из корня репозитория
docker compose -f docker-compose.stt.yml up -d --build

# логи (первый старт качает модель — подождите)
docker compose -f docker-compose.stt.yml logs -f

# остановить
docker compose -f docker-compose.stt.yml down
```

После старта откройте тест-страницу: `http://SERVER_IP:8000/`
(локально — `http://localhost:8000/`).

> Порт 8000 должен быть открыт в фаерволе/таймвебе только для теста.
> На проде доступ к сервису лучше закрыть и ходить только из внутренней сети.

## Проверка здоровья

```bash
curl http://localhost:8000/health
```

## Транскрипция через curl

```bash
curl -F "file=@/path/to/audio.webm" http://localhost:8000/transcribe
```

## Настройка скорости и качества (env)

Задаются в `docker-compose.stt.yml` или через `.env`:

| Переменная | По умолчанию | Описание |
|---|---|---|
| `WHISPER_MODEL` | `small` | `tiny` < `base` < `small` < `medium` (точнее/медленнее) |
| `WHISPER_COMPUTE_TYPE` | `int8` | для CPU оптимально `int8` |
| `WHISPER_DEVICE` | `cpu` | `cuda` при наличии GPU |
| `WHISPER_LANGUAGE` | `ru` | `auto` для автоопределения |
| `WHISPER_BEAM_SIZE` | `1` | `1` = быстро (жадно), `5` = точнее/медленнее |
| `WHISPER_VAD_FILTER` | `true` | пропуск тишины — ускоряет диалоги с паузами |
| `WHISPER_CPU_THREADS` | `0` | `0` = авто; на N ядер можно задать N |

### Рекомендации по железу

- **1 ядро (текущий сервер):** `small` + `beam=1` для теста качества. Скорость низкая —
  тестируйте на коротких записях (1–3 мин).
- **4 ядра:** `small`/`medium`, ускорение примерно в 4 раза.
- **GPU:** `WHISPER_DEVICE=cuda`, `WHISPER_COMPUTE_TYPE=float16`, модель `medium`/`large-v3`.

## Как читать метрики на тест-странице

- **Реалтайм-фактор `xN`:** сколько секунд обработки на 1 секунду аудио.
  `x1.0` = обработка равна длительности; `x0.5` = вдвое быстрее реального времени;
  `x2.0` = вдвое медленнее. Цель для комфорта — ниже `x1.0`.
