# Docker Контейнеризация: Doctor Opus (Medical AI Assistant)

Этот проект настроен для работы в Docker, что гарантирует наличие всех зависимостей для Node.js и Python (включая библиотеки для работы с DICOM и медицинскими изображениями).

## Быстрый запуск (рекомендуется)

Используйте Docker Compose для автоматической сборки и запуска:

```bash
# 1. Создайте файл .env (если его нет) и добавьте API ключи
echo "OPENROUTER_API_KEY=ваш_ключ" >> .env
echo "ASSEMBLYAI_API_KEY=ваш_ключ" >> .env

# 2. Запустите контейнер
docker-compose up -d --build
```

Приложение будет доступно по адресу: `http://localhost:3000`

## Ручная сборка образа

Если вы хотите собрать образ вручную:

```bash
# Сборка
docker build -t doctor-opus-app .

# Запуск
docker run -p 3000:3000 --env-file .env doctor-opus-app
```

## Что внутри образа?

1.  **Node.js 18:** Для работы Next.js фронтенда.
2.  **Python 3.10:** Для работы скриптов обработки данных (DICOM, PDF, транскрипция).
3.  **Системные библиотеки:**
    *   `libcairo2`, `libpango`, `libjpeg` и др. — необходимы для работы графических библиотек (Canvas).
    *   `libgl1` — необходима для работы OpenCV и обработки изображений.
4.  **Python пакеты:** `pydicom`, `numpy`, `Pillow` и другие из `requirements.txt`.

## Устранение неполадок

Если скрипты обработки DICOM не находят файлы, убедитесь, что пути в приложении соответствуют структуре внутри контейнера (`/app/uploads`). В `docker-compose.yml` уже настроены тома (volumes) для сохранения загруженных файлов между перезапусками.
