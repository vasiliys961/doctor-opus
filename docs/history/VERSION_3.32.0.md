# Medical Assistant v3.32.0 - Clinical Context Update
Дата: 27 декабря 2025

## Основные изменения:

### 1. Добавлено окно клинического контекста
Во всех разделах анализа изображений (Общий анализ, ЭКГ, Рентген, КТ, МРТ, УЗИ, Дерматоскопия) и лабораторных данных добавлено текстовое поле для ввода клинического контекста.
- **Цель**: Позволить врачу передать жалобы пациента, анамнез или конкретные диагностические задачи.
- **Реализация**: Новое состояние `clinicalContext` на фронтенде передается через `FormData` в API.

### 2. Обновление бэкенда и библиотек
- **lib/openrouter.ts**: Все функции анализа теперь принимают необязательный параметр `clinicalContext`.
- **lib/openrouter-streaming.ts**: Стриминговые функции также поддерживают передачу контекста.
- **Промпты**: Если контекст предоставлен, он добавляется в начало запроса к LLM в выделенном блоке `=== КЛИНИЧЕСКИЙ КОНТЕКСТ ПАЦИЕНТА ===`.

### 3. Улучшение точности анализа
- Передача контекста значительно снижает вероятность галлюцинаций LLM и помогает сфокусироваться на важных для врача деталях снимка или анализа.
- Система теперь знает, "что искать" на изображении, основываясь на клинике пациента.

### 4. Поддержка в лабораторном анализе
- Контекст добавлен как для одиночных файлов, так и для многостраничных PDF-отчетов.

## Технические детали:
- Изменено 13 файлов (библиотеки, API роуты, страницы фронтенда).
- Сохранена полная обратная совместимость (поле контекста опционально).
- Созданы резервные копии всех затронутых файлов в `backups/clinical_context_backup_20251227/`.
