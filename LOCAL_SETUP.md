# Настройка локальной версии Next.js

## Быстрый старт

1. **Установите зависимости:**
   ```bash
   npm install
   ```

2. **Создайте файл `.env.local`** с переменными окружения:
   ```bash
   cp .env.local.example .env.local
   ```
   
   Затем откройте `.env.local` и заполните реальными ключами:
   ```
   OPENROUTER_API_KEY=sk-or-v1-ваш-ключ
   ASSEMBLYAI_API_KEY=ваш-ключ
   ```

3. **Запустите локальный сервер:**
   ```bash
   npm run dev
   ```

4. **Откройте браузер:**
   ```
   http://localhost:3000
   ```

## Проверка переменных окружения

Если API не работает локально, проверьте:

1. **Файл `.env.local` существует** и содержит правильные ключи
2. **Перезапустите сервер** после изменения `.env.local`:
   ```bash
   # Остановите сервер (Ctrl+C) и запустите снова
   npm run dev
   ```

3. **Проверьте консоль сервера** - там должны быть логи вызовов API

## Отладка

Если ответы не приходят:

1. Откройте консоль браузера (F12) и проверьте ошибки
2. Проверьте консоль терминала где запущен `npm run dev`
3. Убедитесь, что переменные окружения загружены:
   - В коде используется `process.env.OPENROUTER_API_KEY`
   - Next.js автоматически загружает `.env.local`

## Отличия от Vercel

- **Локально:** переменные из `.env.local`
- **Vercel:** переменные из Dashboard → Settings → Environment Variables

Оба должны содержать одинаковые ключи!
